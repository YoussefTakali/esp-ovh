Main workflow
Academic Structure -> Repository Creation -> Student Commits/Branches -> Activity Tracking -> AI-assisted Analysis -> Teacher Review -> Feedback and Grading

Approach and Methodology (long version)
This research uses a Design Science workflow with human-validated evaluation.
The process starts from the academic structure (roles, classes, groups, tasks), then automates repository creation and linking for project teams.
Students generate software traces through commits and branches.
These traces are continuously tracked (activity, deadlines, attempts, progress) to provide objective evidence of engagement and contribution.
An AI-assisted analysis layer transforms these traces into actionable indicators (participation signals, delay risk, contribution visibility), without replacing pedagogical decision-making.
Teachers then perform expert review, contextualize the evidence, and provide final feedback and grading.
The methodology combines technical automation, academic traceability, and responsible evaluation with a human-in-the-loop model.

Approach and Methodology (short version)
Seven-step methodology: academic structuring, repository creation, student commits/branches, activity tracking, AI-assisted analysis, teacher review, and feedback plus grading.
This pipeline improves traceability, reduces manual workload, and keeps final assessment decisions under teacher control.
